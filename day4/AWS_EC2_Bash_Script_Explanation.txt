AWS EC2 AUTOMATION WITH BASH SCRIPT - DETAILED EXPLANATION

PROJECT OVERVIEW This project automates the creation of an Amazon EC2
instance using a Bash shell script and the AWS CLI.

Workflow: 1. Check whether AWS CLI is installed. 2. Install AWS CLI
automatically if missing. 3. Read AWS configuration variables. 4. Create
an EC2 instance. 5. Wait until the instance is in the running state. 6.
Print the created Instance ID.

PREREQUISITES - AWS Account - IAM Role or AWS CLI credentials - AWS CLI
v2 - curl - unzip - Existing EC2 Key Pair - Valid AMI - Valid Subnet -
Security Group(s)

VARIABLES AMI_ID Amazon Machine Image used to launch the EC2 instance.

INSTANCE_TYPE Example: t2.micro, t3.micro, t4g.micro.

KEY_NAME Existing EC2 key pair name.

SUBNET_ID Subnet where the instance is launched.

SECURITY_GROUP_IDS One or more Security Group IDs. They must belong to
the same VPC as the subnet.

INSTANCE_NAME Name tag applied to the EC2 instance.

FUNCTIONS

1.  check_awscli() Checks whether AWS CLI exists using: command -v aws
    Returns failure if AWS CLI is missing.

2.  install_awscli()

-   Updates apt packages.
-   Installs curl and unzip.
-   Downloads AWS CLI v2.
-   Extracts the archive.
-   Runs the installer.
-   Verifies installation.
-   Removes temporary files.

3.  create_ec2_instance() Uses: aws ec2 run-instances

Important options: –image-id –instance-type –key-name –subnet-id
–security-group-ids –tag-specifications –query –output text

Stores the returned Instance ID.

4.  wait_for_instance() Checks the instance state until it becomes
    ‘running’.

Recommended command: aws ec2 wait instance-running –instance-ids
INSTANCE_ID

AWS COMMANDS USED aws –version aws ec2 run-instances aws ec2
describe-instances aws ec2 wait instance-running aws sts
get-caller-identity

COMMON ERRORS FIXED 1. Bash syntax error in if statement. 2. Typo:
security_groupe_ids -> security_group_ids. 3. AWS CLI not installed. 4.
unzip not installed. 5. Typo: –quary -> –query. 6. NoCredentials ->
attach IAM role or configure credentials. 7. Invalid AMI -> use a public
AMI in the correct region. 8. InvalidParameterCombination -> ensure
security groups and subnet belong to the same VPC. 9. Wrong command:
describe-instances-instance-ids Correct: describe-instances
–instance-ids
